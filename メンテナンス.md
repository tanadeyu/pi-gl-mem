# pi-local-mem メンテナンスガイド（2026-06-20時点）

## 3. ディレクトリ構造（仕様書10§3）
```
.pi-local-mem/
  ├─ MEMORY_local.md            # 長期記憶（決定・仕様・durable facts）※自動ロード
  ├─ SCRATCHPAD_local.md        # チェックリスト（後でやること）※手動read
  ├─ daily_local/YYYY-MM-DD.md  # 日次ログ（今日+昨日を自動ロード）
  ├─ notes_local/               # ノート類（手動read）
  └─ pi_memory_local.json         # injectLocal / injectGlobal（デフォルト両方true）
```
グローバル(pi-mem)と完全独立。8ファイルは一切交差しない（実証済み）。

## 4. ツール構成（4ツール・pi-mem互換）
| ツール | 互換先 | 主な機能 |
|---|---|---|
| `write_local_memory` | memory_write | target: long_term/daily/note, mode: append/overwrite, タイムスタンプ+セッションID自動付与 |
| `read_local_memory` | memory_read | target: long_term/scratchpad/daily/note/list |
| `local_scratchpad` | scratchpad | action: add/done/undo/clear_done/list |
| `search_local_memory` | memory_search | 大文字小文字無視の部分一致grep |

## 5. 確立済みの方針
- **方式B（独立実装）**: pi-memへのimport依存ゼロ・必要な関数は内部にコピー持参
- **`_local` postfix**: pi-memとの命名衝突を完全回避
- **injectGlobal**: `false`で`<pi-mem-injected>`除去（後段contextフィルタ・packages順序でpi-memが前であること）
- **デフォルト**: injectLocal/injectGlobal 両方 `true`（勝手に消さない）
- **除外機能**: dashboard/LLM要約/session scanner/git autocommit/catchup/contextFiles自動ロード（重機能は持たない）

## 8. 日常運用
### 8.1 起動確認
`pi` 起動時に以下が表示されれば正常：
```
[Extensions]
  @haha1903/pi-mem, pi-local-mem.ts
```
4ツール衝突エラーが出なければOK。

### 8.2 injectGlobal の切り替え
`.pi-local-mem/pi_memory_local.json` を編集：
```json
{"injectLocal": true, "injectGlobal": false}
```
- `false`: `<pi-mem-injected>` が後段フィルタで除去される（ローカル専用表示）
- `true`: グローバル+ローカル両方の注入を表示（デフォルト）
- 前提: settings.json の packages 順序で pi-mem が pi-local-mem より前であること

### 8.3 データ保護
- `pi uninstall pi-local-mem` は `.pi-local-mem/` を削除しない（データ保護仕様）
- データを完全リセットしたい場合は `.pi-local-mem/` を手動削除 → 次回起動でクリーン生成

## 9. トラブルシューティング
### 9.1 4ツール衝突エラー（最頻トラブル）
**症状**: 起動時に以下が表示され拡張が読み込まれない
```
Error: Tool "write_local_memory" conflicts with /tmp/pi-local-mem.ts
```
**原因**: settings.json に旧パス（/tmp等）と新パス（本体）が両方登録されている
**対処**:
```bash
grep local-mem ~/.pi/agent/settings.json
# 旧エントリ（/tmp/... 等）が見えたら直接編集で削除
```
`pi uninstall pi-local-mem` は「No matching package found」で消せないため、settings.json の packages 配列を直接編集する。編集後は `python3 -c "import json; json.load(open('settings.json'))"` で正当性確認。

### 9.2 参照先変更時の手順
インストール先を変えるときは単に `pi install` するだけでなく、**旧エントリが settings.json に残っていないか grep で必ず確認** すること（§9.1のトラブル原因）。

### 9.3 仕様書↔本体の不一致確認
```bash
# 仕様書10 §8コードブロックと ts 本体をdiff
diff <(sed -n '/§8/,/§9/p' pi-local-mem仕様書10.md) pi-local-mem.ts
# 差分ゼロが正常
```

## 10. 教訓（重複登録トラブルより）
- `pi install` はパス単位で登録される。異なるパス（/tmp と /mnt/c/...）は別エントリとして**両方残る**
- `pi uninstall <名前>` はパッケージ名解決が必要で、ローカルファイル参照の場合「No matching package found」になり**消せない**
- 重複除去は **settings.json の packages 配列を直接編集** するのが確実
- インストール先を変えるときは「uninstall→install」手順だけでなく、**settings.json に旧エントリが残っていないか grep で必ず確認** すること
