# 作業手順（pi-gl-mem v1.0.0）

## 概要
pi-local-mem を改名＋global対応した新パッケージ。
- global: `~/.pi/agent/pi-gl-mem/MEMORY.md`（1ファイルのみ・注入・書き込み可）
- local: `./.pi-gl-mem/`（4ファイル+4ツール・従来のpi-local-mem形式そのまま）
- 互換性: あり（.pi-local-mem/ の内容をリネーム移行可能）
- 配布形式: GitHub リポジトリ（npm publish はしない）

## 完了済み ✅
- ✅ 上探索ロジック追加（cwd→上方向・上限なし）
- ✅ 早期return共通ガード（.pi-gl-mem/なければ何もしない）
- ✅ init.sh 作成（Y/N確認付き）
- ✅ pi-mem（@haha1903/pi-mem）アンインストール + グローバル記憶整理
- ✅ header更新（v1.1.0→改名後v1.0.0に変更予定）
- ✅ ツール個別チェック削除（早期returnに統一）

## やること（順番通り）

### Phase 1: 改名（pi-local-mem → pi-gl-mem）
1. **pi-gl-mem.ts を新規作成**（pi-local-mem.ts をコピーして修正）
   - 全 `pi-local-mem` → `pi-gl-mem`
   - 全 `.pi-local-mem/` → `.pi-gl-mem/`
   - header: v1.0.0（新規）
   - 各ツール名: `write_local_memory` 等は維持（内部パスが変わるだけ）

2. **package.json 新規作成**
   - `name`: `@tanadeyu/pi-gl-mem`
   - `pi.extensions`: `["./pi-gl-mem.ts"]`
   - `bin`: `{"pi-gl-mem-init": "./pi-gl-mem-init.sh"}`

3. **pi-gl-mem-init.sh 更新**
   - `.pi-local-mem/` → `.pi-gl-mem/` を作成

4. **旧ファイル整理**
   - `pi-local-mem.ts` は削除 or リネーム（新規作成のため）

### Phase 2: global対応追加
5. **global用パス追加**
   ```typescript
   const globalDir = path.join(os.homedir(), '.pi', 'agent', 'pi-gl-mem');
   const globalMemoryFile = path.join(globalDir, 'MEMORY.md');
   ```
   - globalは MEMORY.md 1ファイルのみ（daily/notes/scratchpadなし）

6. **global MEMORY.md 注入**
   - `before_agent_start` で読んで systemPrompt に追記
   - ファイルがなければスキップ

7. **global用ツール追加（2つ）**
   - `write_global_memory`（MEMORY.md に追記 or 上書き）
   - `read_global_memory`（MEMORY.md 内容を返す）

8. **ツール名整理**
   - global: `write_global_memory` / `read_global_memory`
   - local: 現状の4ツールを維持（`write_local_memory` / `read_local_memory` / `local_scratchpad` / `search_local_memory`）

### Phase 3: ローカルテスト
9. **ローカルでテスト**（公開前に動作確認）
    ```bash
    # 現在のリポジトリに pi-gl-mem.ts を配置
    pi install /mnt/c/projects/pi-local-mem
    # または github 形式のテスト
    ```

10. **3つの動作テスト（約15分）**
    | テスト | 内容 |
    |--------|------|
    | init | 未init→何もしない→pi-gl-mem-init→作成→動作 |
    | 上探索 | サブdirから起動→上の.pi-gl-mem/認識 |
    | global書込 | write_global_memoryでglobal MEMORY.md追記→再起動で反映 |

### Phase 4: GitHub公開
11. **GitHub に push**（今のリポジトリ内で管理）
    ```bash
    git add pi-gl-mem.ts pi-gl-mem-init.sh
    git commit -m "add pi-gl-mem v1.0.0"
    git push
    ```

12. **インストール**
    ```bash
    pi install github:tanadeyu/pi-local-memo0
    ```
    ※ 同じリポジトリ内に pi-local-mem.ts と pi-gl-mem.ts が共存する形

## 注意
- pi 再起動が必要（変更後に反映）
- `.pi-local-mem/` との互換性なし（あれば無視・新しいプロジェクトでは .pi-gl-mem/ を使う）
- バージョンは v1.0.0（pi-local-mem v1.0.0 とは別物）
- **npm publish はしない**。GitHub リポジトリ経由で配布
