# パッケージ配布の方法
pi-local-mem.ts を `@haha1903/pi-mem 1.0.1` のようにパッケージとしてインストール可能にするための手順をまとめる。現状の `pi install ./pi-local-mem.ts`（ローカルファイル直接参照）から、名前＋バージョン指定でインストールできる形式に変更する方法。

## 前提知識：pi 拡張の読み込み方式

| 方式 | インストール方法 | settings.json 登録例 | バージョン指定 |
|---|---|---|---|
| ローカルファイル参照 | `pi install ./pi-local-mem.ts` | `"./pi-local-mem.ts"`→絶対パス化 | 不可（ヘッダーコメントのみ） |
| npm パッケージ | `pi install @haha1903/pi-mem` | `"@haha1903/pi-mem"` | `1.0.1` と明示可 |
| GitHub リポジトリ直 | `pi install https://github.com/...` | URL文字列 | タグ/ブランチ指定可 |
| ローカルディレクトリ | `pi install ./pkg-dir/` | `"./pkg-dir/"`→絶対パス化 | 不可 |

現在の pi-local-mem.ts は最上行の方式。以降はこれらをパッケージ化する手順。

## 方法 A：npm 公開（本格的）

### 手順
1. `pi-local-mem.ts` と同階層に `package.json` を作成
2. npm に公開（`npm publish --access public`）
3. インストール: `pi install @tanadeyu/pi-local-mem`

### package.json 必要項目

```json
{
  "name": "@tanadeyu/pi-local-mem",
  "version": "1.0.0",
  "description": "Project-local memory for pi coding agent — independent, self-contained, zero dependencies",
  "main": "pi-local-mem.ts",
  "type": "module",
  "keywords": ["pi-package", "pi-extension"],
  "pi": {
    "extensions": ["./pi-local-mem.ts"]
  },
  "publishConfig": { "access": "public" },
  "peerDependencies": {
    "@earendil-works/pi-coding-agent": "*",
    "@sinclair/typebox": "*"
  }
}
```

`"pi": { "extensions": [...] }` が pi にどのファイルが拡張かを伝える必須フィールド。`version` を上げて再公開すれば、インストール先で `pi update` で更新可能。

### スコープ名の選択
- `@tanadeyu/pi-local-mem` — 個人スコープ。npm publish に npm アカウント＋スコープ作成が必要
- `pi-local-mem` — スコープなし。全世界で名前が衝突しない限り使えるが、`@haha1903/pi-mem` のような既存パッケージと命名規則を揃えるならスコープ推奨

## 方法 B：GitHub リポジトリ直指定（npm publish 不要）

### 手順
1. `pi-local-mem.ts` と同階層に `package.json` を作成（方法Aと同内容）
2. GitHub に push
3. インストール: `pi install https://github.com/tanadeyu/pi-local-memo0`

### 特徴
- npm publish が不要。GitHub に `package.json` を含むファイルがあるだけで動作する
- バージョン指定: リポジトリルートの `package.json` の `version` が使われる
- 更新: GitHub に push → インストール先で `pi install` し直す（上書き登録される）
- `pi install` の引数に `#branch` や `#tag` を付けて特定バージョンを指定できるかは要検証

## 方法 C：ローカルディレクトリ参照（npm 非公開・簡易版）

### 手順
1. サブディレクトリ（例: `pkg/`）を作成
2. その中に `package.json` と `pi-local-mem.ts` を配置
3. インストール: `pi install ./pkg/`

現在の `pi install ./pi-local-mem.ts` と実質同じだが、ディレクトリ単位にすることで将来的なファイル分割（`lib.ts` の分離など）に対応できる。

## 比較

| 観点 | A: npm 公開 | B: GitHub 直指定 | C: ローカルディレクトリ |
|---|---|---|---|
| 手間 | 大（npm publish・version管理） | 中（package.json追加＋push） | 小（package.json追加のみ） |
| 公開範囲 | 全世界 | GitHub知ってる人のみ | 自分のPCのみ |
| バージョン管理 | ○（semver公式） | △（package.jsonのversionのみ） | △（同上） |
| 依存解決 | ○（peerDependencies） | ○（同上） | ○（同上） |
| 他PCで使う | ○（npm install） | ○（git clone＋pi install） | × |
| ファイル分割 | ○ | ○ | ○ |

## 推奨

現状の `pi install ./pi-local-mem.ts` は機能的に完全に動作している。パッケージ化する優先度は低い。変更する場合の最初のステップとしては、**方法B（package.json追加→GitHub直指定）** が最も少ない手間で名前＋バージョン指定を実現できる。

方法A（npm公開）に進む場合は、以下の追加作業が必要:
- npm アカウント作成（未の場合）
- スコープ作成（`@tanadeyu`）
- `npm login` / `npm publish --access public`
- 更新のたびに `npm version patch` → `npm publish`
